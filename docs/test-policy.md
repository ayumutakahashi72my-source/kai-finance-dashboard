# テスト方針

## 書くテスト（3点のみ）

```
src/__tests__/score.test.ts        スコア計算・グレード境界値
src/__tests__/csv.test.ts          重複検知・source_hash生成
src/__tests__/category-rag.test.ts RAGヒット判定・normalizeKeyword
```

これ以外のテストは書かない。E2E・コンポーネントテストも書かない。

## 書かない理由

- 個人プロジェクト規模では費用対効果が合わない
- 上記3点はビジネスロジックの中核で、ここだけ守れば致命傷を避けられる
- UIはVitestでなく実機確認で十分

## 実行タイミング

- 機能実装完了時
- 他機能から影響を受ける変更時
- Phase完了時

## 追加していい場合

「新しいビジネスロジックの中核」を追加した場合のみテストを増やしてよい。
UIや単純なCRUDのためにテストを増やさない。
